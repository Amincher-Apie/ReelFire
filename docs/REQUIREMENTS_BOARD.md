# ReelFire 需求看板

> 看板只使用角色称谓。组内成员与角色的真实映射由内部表维护。

## 1. 状态定义

| 状态 | 含义 |
|---|---|
| `BACKLOG` | 已记录但不进入当前 MVP |
| `TODO` | 需求明确，可以开始 |
| `DOING` | 已由负责人领取并正在开发 |
| `REVIEW` | 已完成开发，等待代码审查或联调 |
| `DONE` | 已通过验收并保留证据 |
| `BLOCKED` | 存在明确阻塞，必须填写原因 |

状态变更规则：负责人更新状态；只有在代码、文档或测试证据可检查时才能进入 `DONE`。接口或 JSON 字段变更须先经组长确认。

## 2. 当前迭代目标

```text
FPS 视频上传
  -> 异步任务
  -> OpenCV + YOLO + 三项评分
  -> 关键帧与单个推荐片段
  -> 人工审核和边界调整
  -> FFmpeg 16:9 / 9:16 输出
  -> JSON 报告与历史重开
```

## 7. 看板更新规则

1. 每次阶段检查前由负责人更新自己的任务；
2. `DOING` 任务应尽量每个角色不超过 2 个；
3. `BLOCKED` 必须写原因、影响和解除方案；
4. 需求变更先更新 PRD，再调整看板；
5. `DONE` 必须附代码、测试、截图或文档证据；
6. 15:30 功能冻结后，P1 不得进入 `DOING`。
