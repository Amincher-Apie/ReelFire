# ReelFire 四天需求与进度看板

> 课程题目：1. 基于 YOLO+Agent 的智能数字媒体内容理解系统。
>
> 公共仓库只使用角色称谓；私有人员映射在仓库外维护。本看板只在代码、测试、截图或文档证据可检查时标记 `DONE`。

## 1. 状态定义

| 状态 | 含义 |
|---|---|
| `BASELINE_DONE` | 当前仓库已经实现并通过现有验收，但不代表整个课程题目已完成 |
| `TODO` | 课程补齐任务已明确，可以开始 |
| `DOING` | 负责人正在开发 |
| `REVIEW` | 已提交，等待代码审查、联调或验收 |
| `DONE` | 已通过课程口径验收并保留证据 |
| `BLOCKED` | 有明确阻塞，必须写原因、影响和解除方案 |

## 2. 当前代码基线

```text
视频上传
  -> Flask 异步任务
  -> OpenCV + YOLO11n + 三项评分
  -> 关键帧、检测框和候选片段
  -> 人工 keep/skip、标签、备注和边界调整
  -> FFmpeg 粗剪
  -> JSON 报告与历史重开
```

## 6. 更新规则

1. 每次检查点前由主责人更新状态、证据和完成比例；
2. 每名成员同时 `DOING` 的任务不超过 2 个；
3. `BLOCKED` 必须记录原因、影响、临时方案和解除时间；
4. 字段变化先更新 `API.md`/Schema，再修改调用方；
5. `DONE` 必须有代码、测试/截图和文档三类证据；
6. Day 4 12:00 后不新增功能，只修阻塞缺陷；
7. `.codegraph/`、模型、输出视频、密钥和本地数据库不得误提交。
