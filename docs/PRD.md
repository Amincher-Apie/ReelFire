# PRD - 智能视频精彩片段提取系统

## 1. 项目概述

### 1.1 项目名称
CV 综合项目实战：智能视频精彩片段提取系统

### 1.2 业务目标
面向游戏录屏、游戏场景演示和宣传素材，利用目标检测、画面变化和运动强度，自动找出值得人工复核的关键帧和候选片段。

### 1.3 技术栈
- Python、Flask、HTML/CSS/JavaScript、OpenCV、YOLO、FFmpeg

## 3. 评分规则

### 3.1 评分公式

```
highlight_score =
    object_score * 0.45
  + scene_change_score * 0.35
  + motion_score * 0.20
```

### 3.2 指标说明

| 指标 | 含义 | 计算方式 |
|---|---|---|
| object_score | 当前采样帧检测到的目标数量归一化分数 | min(count / 20, 1.0) |
| scene_change_score | 当前帧与相邻采样帧的画面差异 | 相邻帧灰度差异均值归一化 |
| motion_score | 画面运动强度或连续帧变化程度 | 光流法计算运动幅度 |
| highlight_score | 三项指标加权后的最终推荐分数 | 加权求和 |

## 4. 推荐片段边界要求

```
0 <= start < end <= video_duration
```

### 4.1 异常场景处理

- 最高分关键帧靠近视频开头：从开头截取到片段结束
- 最高分关键帧靠近视频结尾：从片段开始截取到结尾
- 用户设置的目标时长大于原视频时长：返回整个视频
- 视频没有音频轨道：正常处理，输出无声视频
- 关键帧数量不足：返回所有可用关键帧
- 连续多个高分帧来自同一个镜头：去重合并

## 5. 验收标准

### 5.1 功能验收

| 编号 | 验收项 | 通过条件 |
|---|---|---|
| ACC-001 | 视频上传 | 成功上传视频并创建任务 |
| ACC-002 | 任务状态 | 状态能从 created -> running -> completed/failed |
| ACC-003 | 目标检测 | YOLO 能检测到视频中的目标 |
| ACC-004 | 评分计算 | 能正确计算精彩分数 |
| ACC-005 | 片段推荐 | 能推荐至少一个精彩片段 |
| ACC-006 | 粗剪输出 | 能生成粗剪视频文件 |
| ACC-007 | 报告输出 | 能输出 JSON 格式的分析报告 |
| ACC-008 | 历史任务 | 能从历史记录重新打开任务 |

### 5.2 异常验收

| 编号 | 验收项 | 通过条件 |
|---|---|---|
| ERR-001 | 格式错误 | 给出明确错误提示 |
| ERR-002 | 空文件 | 给出明确错误提示 |
| ERR-003 | 模型不存在 | 给出明确错误提示 |
| ERR-004 | 任务失败 | 状态标记为 failed，保存错误信息 |
