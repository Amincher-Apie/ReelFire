# ReelFire 验收清单

> 验收素材：至少一段 CS:GO/CS2 或 Valorant 对战录像；同时准备有音频和无音频样本。

## 1. 硬性完成条件

- [ ] 可按照 README 从新环境启动项目；
- [ ] 页面可以上传合法 FPS 视频并创建任务；
- [ ] 上传后立即返回任务编号，不等待完整推理；
- [ ] 页面可以观察任务状态变化；
- [ ] 至少一个 OpenCV、YOLO 或运动分析功能真实运行；
- [ ] 结果文件可打开，字段和分数可解释；
- [ ] 失败状态具有明确错误信息；
- [ ] 历史任务可以重新打开；
- [ ] 文档、截图、测试和 Bug 记录齐全。

任意一项不满足，项目不能标记为完成。

## 2. 环境与启动

- [ ] `python setup_environment.py` 可完成安装或复用环境；
- [ ] `python -m pip check` 返回 `No broken requirements found`；
- [ ] `torch.cuda.is_available()` 在 NVIDIA 演示机返回 `True`；
- [ ] `ffmpeg -version` 和 `ffprobe -version` 正常；
- [ ] `python app.py --host 127.0.0.1 --port 7880` 可启动；
- [ ] `GET /api/health` 返回 `status: ok`；
- [ ] 健康接口能报告模型、GPU 和 FFmpeg 状态。

## 3. 上传与任务状态

- [ ] MP4 对战录像上传成功；
- [ ] 返回内容包含 `ok: true`、`job_id` 和状态；
- [ ] `job_id` 唯一且只能映射到合法任务目录；
- [ ] 原始素材保存到任务 `input/`；
- [ ] 状态按 `created -> queued -> running -> completed/failed` 变化；
- [ ] 页面显示项目名、素材名、创建/开始/结束时间；
- [ ] 失败任务仍保留 `job.json`；
- [ ] 运行中的任务不能直接删除。

## 4. FPS 视频分析

- [ ] OpenCV 正确读取时长、分辨率和 FPS；
- [ ] 采样按时间间隔执行，报告保存采样参数；
- [ ] 真实 YOLO 模型至少完成一次推理；
- [ ] 检测结果包含类别、置信度和边界框；
- [ ] `object_score` 在 `[0,1]`；
- [ ] `scene_change_score` 在 `[0,1]`；
- [ ] `motion_score` 在 `[0,1]`；
- [ ] 综合分数符合 `0.45/0.35/0.20` 权重；
- [ ] 报告保存三项分数、权重和最终分数；
- [ ] 至少一组真实 FPS 素材产生合理关键帧。

## 5. 关键帧和候选片段

- [ ] 关键帧按综合分数筛选；
- [ ] 相邻高分帧经过最小时间间隔去重；
- [ ] 关键帧图片可在页面打开；
- [ ] 可以选择保留或忽略；
- [ ] 可以调整顺序；
- [ ] 可以添加标签和备注；
- [ ] 刷新页面后人工修改仍然存在；
- [ ] 自动生成至少一个候选片段；
- [ ] 后端保证 `0 <= start < end <= video_duration`；
- [ ] 用户可以修改开始和结束时间；
- [ ] 非法边界被拒绝并给出可读错误。

## 6. 边界专项测试

- [ ] 最高分关键帧靠近视频开头；
- [ ] 最高分关键帧靠近视频结尾；
- [ ] 目标时长大于原视频时长；
- [ ] 目标时长等于原视频时长；
- [ ] 视频时长很短但仍可生成合法片段；
- [ ] 关键帧数量不足；
- [ ] 多个高分帧来自同一个镜头；
- [ ] 用户输入 `start < 0`；
- [ ] 用户输入 `end > video_duration`；
- [ ] 用户输入 `start >= end`。

## 7. FFmpeg 输出

- [ ] 16:9 MP4 可以生成并播放；
- [ ] 9:16 MP4 可以生成并播放；
- [ ] 分辨率和画面比例符合规格；
- [ ] 有音频源视频的输出可听到原音频；
- [ ] 无音频源视频不会导致任务失败；
- [ ] 靠近结尾的推荐仍尽量保持目标时长；
- [ ] 输出文件路径记录在报告中；
- [ ] 关键帧联系表可以打开。

## 8. 页面状态

- [ ] 初始空状态清晰；
- [ ] 上传中有禁用或进度提示；
- [ ] `queued/running` 有加载状态；
- [ ] 无关键帧时不出现 JavaScript 错误；
- [ ] 任务失败显示后端返回的可读错误；
- [ ] 任务完成显示关键帧、分数和输出；
- [ ] 历史任务列表为空、加载中、失败和完成状态完整；
- [ ] 页面不会因接口参数错误崩溃。
