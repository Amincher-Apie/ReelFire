# ReelFire 前端交互说明

> 前端工程师 · 击杀集锦提取工具 · 霓虹竞技风格

## 一、页面总览

系统为**单页应用**，包含三个视图，通过顶栏导航切换：

| 视图 | 用途 | 入口 |
|---|---|---|
| 上传素材 | 创建新的分析任务 | 顶栏导航「上传素材」 |
| 任务记录 | 查看/管理所有历史任务 | 顶栏导航「任务记录」 |
| 审核详情 | 查看结果 + 人工复核 | 从任务记录点击「查看」进入 |

## 二、用户主流程

1. 用户在「上传素材」填写项目名、选择游戏、拖拽视频、设置参数
2. 点击创建 POST /api/jobs，获得 job_id 后跳转到任务记录
3. 任务状态从 queued -> running -> completed/failed(自动轮询)
4. 状态为 completed 时，点击「查看」进入审核详情
5. 查看评分条、候选片段、关键帧网格
6. 对关键帧执行保留/忽略、排序、打标签、写备注
7. 调整片段开始/结束时间
8. 点击「保存审核结果」写入 JSON
9. 点击「生成粗剪视频」触发 FFmpeg 输出
10. 查看视频播放器、联系表、JSON 报告

## 三、视图详解

### 3.1 上传素材视图

布局：单个卡片，从上到下依次为：

1. 项目名称(文本输入)
2. 游戏类型(下拉: CS:GO/CS2 / Valorant / 其他FPS)
3. 视频文件(拖拽上传区 + 点击选择)
4. 采样间隔(数字输入, 默认0.5s, 范围0.25-2.0)
5. 目标时长(数字输入, 默认30s, 范围5-300)
6. 输出规格(下拉: 16:9 / 9:16)
7. 创建任务按钮 + 重置按钮

交互行为：

| 操作 | 反馈 |
|---|---|
| 拖拽文件到上传区 | 边框变红 + 背景变亮 |
| 移开拖拽 | 恢复原状 |
| 选择文件后 | 显示文件名和大小 |
| 必填项未填提交 | 浏览器原生校验提示 |
| 不支持的格式 | 后端返回错误, 前端展示 |
| 点击创建 | 按钮变「上传中...」并禁用 |
| 上传成功 | 绿色Toast + 自动跳转任务记录 |
| 上传失败 | 红色Toast, 按钮恢复 |
| 点击重置 | 清空所有输入 |

状态覆盖: 初始空白(拖拽区+提示) -> 文件已选(文件名+大小) -> 上传中(按钮禁用) -> 上传失败(红色提示)

### 3.2 任务记录视图

布局：表格 + 三种兜底状态

| 列 | 内容 |
|---|---|
| 任务编号 | 等宽字体显示 job_id |
| 项目名称 | 用户填写的项目名 |
| 素材名称 | 上传的文件名 |
| 游戏 | CS:GO/CS2 / 无畏契约 / 其他FPS |
| 状态 | 彩色标签 + 圆点(五种颜色) |
| 创建时间 | 本地化格式 |
| 操作 | 查看 + 删除按钮 |

状态标签颜色：

| created | queued | running | completed | failed |
|---|---|---|---|---|
| 灰色 | 蓝色(#0acefe) | 橙色(#f0a030) | 绿色(#20b06e) | 红色(#ff4655) |

交互行为：

| 操作 | 反馈 |
|---|---|
| 进入视图 | 加载动画 -> 显示数据/空状态/错误 |
| 列表为空 | 空状态插图 + 「新建任务」按钮 |
| 加载失败 | 错误信息 + 「重试」按钮 |
| 点击查看 | 进入审核详情(含loading过渡) |
| 点击删除 | 确认框 -> DELETE请求 |
| 运行中任务 | 删除按钮禁用, hover提示 |
| 右上角新建 | 切换到上传素材视图 |

### 3.3 审核详情视图

按任务状态分 5 种展示形态：

**排队中**: 旋转图标 + "任务已提交, 等待处理...", 自动轮询

**分析中**: 脉冲动画 + "任务正在分析中" + 进度条, 自动轮询

**完成**(完整形态, 从上到下):

1. 视频信息栏: 素材名 / 游戏 / 状态标签 / 创建时间
2. 精彩评分条: 四列(目标/场景变化/运动/综合) + 权重文字
3. 候选片段编辑器: 开始/结束时间输入 + 排序 + 实时时长显示 + 边界校验
4. 关键帧审核网格: 每帧含缩略图/时间戳/分数徽章/四项分数条/保留忽略按钮/标签下拉/排序输入/备注输入
5. 操作区: 保存审核按钮 + 生成粗剪按钮
6. 输出区: 视频播放器 + 联系表链接
7. 报告入口: 查看JSON按钮 -> 模态弹窗
8. 删除按钮(右上角)

**失败**: 红色边框卡片 + 错误消息

**加载中**: 大号旋转图标 + "加载任务信息..."

## 四、关键交互规则

### 轮询
- 仅 queued/running 时启动(每2秒)
- completed/failed 时自动停止
- 切换视图时停止

### 边界校验
- 前端实时: start >= end 时片段卡边框变红
- 后端二次: 保存时验证 0 <= start < end <= duration

### Toast通知
- 右上角, 3.5秒自动消失
- 三类: success(绿) / error(红) / info(蓝)
- 多通知垂直堆叠

### 模态弹窗
- 查看JSON报告时弹出, 点击遮罩或Escape关闭
- 内容为格式化缩进JSON

### 图片容错
- 关键帧图片加载失败显示灰色占位
- 不显示broken icon, 不产生JS错误

## 五、错误边界

1. 所有fetch有try-catch, 不导致页面崩溃
2. 后端返回 ok:false 时展示 error 字段
3. 无数据时显示空状态, 不出现undefined引用错误
4. 非法输入有客户端提示
5. 图片加载失败有占位UI

## 六、API调用时序

POST /api/jobs(上传) -> GET /api/jobs/<id>(轮询状态) -> GET /api/jobs/<id>/report(加载报告) -> PATCH /api/jobs/<id>/review(保存审核) -> POST /api/jobs/<id>/rough-cut(生成视频) -> GET /api/jobs/<id>/report(刷新输出)

## 七、状态覆盖矩阵

| 场景 | 表现 |
|---|---|
| 网络断开 | 任务列表显示错误+重试; 审核页红色Toast |
| 空任务列表 | 空状态插图+引导按钮 |
| 无关键帧 | 显示"暂无关键帧" |
| 图片加载失败 | 灰色占位 |
| 接口异常 | 红色Toast |
| 服务不可用 | 网络错误Toast |
| 刷新页面 | 历史任务重新加载 |
| 重启服务 | 从outputs/恢复数据 |
| 页面之间切换 | 状态保持, 视图动画过渡 |

## 八、对接约束

| 约束 | 说明 |
|---|---|
| 响应格式 | 成功含 ok:true, 失败含 ok:false+error |
| 字段名 | 与 TEAM_GUIDE.md 一致, 不可单方修改 |
| 时间格式 | ISO 8601 带时区 |
| 图片路径 | keyframes[].image 相对于 outputs/<job_id>/ |
| 视频路径 | output.video 相对于 outputs/<job_id>/ |
| CORS | 同源部署, 无需额外配置 |
