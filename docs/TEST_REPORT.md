# 测试报告

## 1. 测试概述

### 1.1 测试目标
验证智能视频精彩片段提取系统的各项功能是否正常工作，确保系统能够正确处理各种输入情况。

### 1.2 测试范围
- 正常视频上传测试
- 格式错误测试
- 空文件测试
- 模型文件不存在测试
- 任务失败状态测试
- 历史任务重新打开测试
- 删除任务测试
- 推荐片段边界测试
- 视频有音频和无音频测试
- 前端状态测试

### 1.3 测试环境
- 操作系统：Windows 10/11
- Python 版本：3.9+
- Conda 环境：yolo
- Flask 版本：2.3.0+
- FFmpeg 版本：8.1.2
- YOLO 模型：yolo11n.pt

## 3. Bug 记录

| 编号 | 问题现象 | 复现步骤 | 原因 | 修复方案 | 验证结果 |
|---|---|---|---|---|---|
| BUG-001 | 粗剪视频在第6秒出现重复画面 | 分析视频并生成粗剪 | 推荐片段重叠（4.0-10.0s 和 8.0-20.0s） | 添加片段合并逻辑，按时间排序后合并重叠区间 | ✅ 已修复 |
| BUG-002 | FFmpeg 报错 "to value smaller than -ss" | 生成粗剪视频 | 击杀帧未按时间戳排序，导致负的片段时长 | 按时间戳排序关键帧后再处理 | ✅ 已修复 |
| BUG-003 | numpy.float32 类型 JSON 序列化错误 | 查看分析报告 | YOLO 检测结果包含 numpy.float32 类型 | 将所有数值转换为 Python float | ✅ 已修复 |
| BUG-004 | 粗剪视频与原视频几乎无变化 | 分析视频并生成粗剪 | 合并了所有推荐片段，未选择最高分片段 | 修改为按分数排序选择片段 | ✅ 已修复 |
| BUG-005 | API 返回 404 错误 | 上传视频时 | Flask debug 模式自动 reload 导致路由未注册 | 关闭 debug 模式（debug=False） | ✅ 已修复 |
| BUG-006 | 评分权重不一致 | 查看分析报告 | __init__ 权重(0.15/0.15/0.10)与计算权重(0.45/0.35/0.20)不符 | 统一权重定义，使用实例属性 | ✅ 已修复 |
| BUG-007 | 片段靠近结尾时只输出几秒 | 最高分帧在视频结尾 | 只支持向后扩展，结尾时无法扩展 | 添加向前扩展逻辑 | ✅ 已修复 |
