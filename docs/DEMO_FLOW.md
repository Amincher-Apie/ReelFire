# ReelFire 8 分钟演示流程

## 1. 演示准备

演示前准备：

- 一段 1～3 分钟、已验证可产生关键帧的 CS:GO/CS2 或 Valorant MP4；
- 一个已经完成的历史任务，用于网络、推理或编码较慢时兜底；
- 一段无音频短视频，用于说明兼容性；
- 16:9 和 9:16 两个已生成输出；
- `job.json`、`analysis_report.json`、关键帧联系表；
- 一份可复现的 Agent 输出、知识库引用和工具调用日志；
- 自定义模型/多阈值对比图、运动轨迹、统计图和 HTML/PDF 报告；
- 正常/异常测试报告和三个真实 Bug 记录；
- 浏览器、终端和文件夹窗口提前打开，关闭无关通知。

## 3. 推荐讲解词

### 开场

> FPS 玩家的一局录像可能持续几十分钟，但真正值得分享的交火、残局和连续击杀只占很短时间。ReelFire 通过目标检测、场景变化和运动强度自动推荐候选，再把最终决定权交给玩家。

### 算法

> 系统不是只给出一个不可解释的“精彩”标签。每张采样帧都有目标分数、场景变化分数和运动分数，并按照 0.45、0.35、0.20 的权重生成综合分数。连续高分帧还会按时间间隔去重，避免同一场交火重复出现。

### 人工复核

> 自动推荐只是初选。玩家可以保留或忽略关键帧、调整排序、增加标签和备注，也可以修改片段边界，确保击杀前的铺垫和击杀后的反馈都被保留。

### Agent 与知识库

> Agent 不直接猜测视频事件，而是先读取真实视觉报告，再检索媒体分类和审核规范，最后生成结构化摘要、标签、建议和三态意见。页面同时展示证据引用与工具调用日志；低置信度或知识库无命中时进入待复核，不把不确定内容说成事实。

### 交付

> 每个任务都有独立目录、状态文件和分析报告。即使任务失败也会保存错误；完成任务可以从历史列表重新打开，并输出横屏、竖屏视频和关键帧联系表。

## 4. 演示兜底策略

| 问题 | 现场处理 |
|---|---|
| YOLO 推理耗时 | 创建任务后切换到预先完成的历史任务，同时说明异步设计 |
| FFmpeg 临时编码慢 | 展示已生成输出，并保留本次任务继续在后台运行 |
| 演示素材没有高分结果 | 使用预先验证素材，不在现场临时更换未知视频 |
| 浏览器状态异常 | 刷新后从历史任务重新打开，体现持久化能力 |
| GPU 不可用 | 健康接口展示 CPU 回退；演示使用预生成结果 |
| 输出无音频 | 同时展示 ffprobe 音轨检测和无音频兼容测试 |
