# 知识库来源与使用边界

本表用于追踪 ReelFire 媒体审核知识库的来源、整理方式和许可边界。仓库只保存团队整理后的规则，不复制课程材料全文，也不包含个人信息。

| 来源 ID | 来源 | 用途 | 整理方式 | 许可/使用边界 |
|---|---|---|---|---|
| `SRC-PROJECT-001` | ReelFire 仓库中的 CV 报告字段、评分逻辑和审核流程 | 确定 Agent 可引用的类别、置信度、时间戳、分数和片段边界 | 由项目字段与现有代码人工归纳 | 项目内部资料；仅用于本课程项目 |
| `SRC-COURSE-001` | 课程任务要求中的 Agent、知识库、审核和可追溯性要求 | 确定三态审核、Top-K 检索、失败降级和证据留存要求 | 仅提炼要求，不复制任务书全文 | 教学使用；不得重新发布原始课程材料 |
| `SRC-RULES-001` | 团队编写的媒体审核规则 | 形成可切分、可向量化和可引用的知识条目 | 对前两项来源进行人工规则化、去重和风险校验 | 团队原创规则，可随项目源代码使用 |

## 更新要求

- 新增条目时必须填写稳定的 `knowledge_id`，并在本表登记来源。
- 不把课程材料、网页或第三方资料的大段原文直接放入知识库。
- 任何外部来源都要记录可核验的标题、链接、许可或教学使用说明。
- 知识条目只解释审核规范，不把未被 CV 检测的 FPS 事件写成事实。
